void Agencia::matchAlugueres ( )

Passa para cada Aluguer da Agencia apontadore para a Viatura que lhe pertence.

Para nao haver redundancia de dados guardados nos ficheiros, no ficheiro dos Alugueres so se guarda os ID's dos Viaturas atribuidas a cada Aluguer. Quando se faz o carragmento dos Alugueres é necessario fazer o "match" dos Alugueres com as Viaturas. Esta funcao vai ao Aluguer buscar esse ID e adiciona-lhe o apontador para a Viatura.

void Agencia::matchClientes ( )

Passa para cada Cliente da Agencia apontadores para os Alugueres que ele tem activos.

Para nao haver redundancia de dados guardados nos ficheiros, no ficheiro dos Clientes so se guarda os ID's dos Alugueres que este tem activo. Quando se faz o carragmento dos Clientes é necessario fazer o "match" dos Clientes com os Alugueres. Esta funcao vai ao Cliente buscar esses ID's e adiciona-lhe os apontadores para os ALugueres.

void Agencia::matchFuncionarios ( )

Passa para cada Funcionario da Agencia apontadores para os Alugueres que ele tem activos.

Para nao haver redundancia de dados guardados nos ficheiros, no ficheiro dos Funcionarios so se guarda os ID's dos Alugueres que este tem activo. Quando se faz o carragmento dos Funcionarios é necessario fazer o "match" dos Funcionarios com os Alugueres. Esta funcao vai ao Funcionario buscar esses ID's e adiciona-lhe os apontadores para os ALugueres.

template<class T >
void Agencia::multiplePagesView ( char &  ch,
int &  x,
int &  y,
int &  numeroObjectos,
const vector< T * > &  objectos 
) const

Funcao template que trata do visionamento de vectores objectos com mais que uma pagina de resultados.

A funcao recebe 2 valores (x e y) que representam a posicao no vector do primeiro e ultimo objecto a apresentar, recebe tambem o numero de objectos que pode representar ao mesmo tempo.
